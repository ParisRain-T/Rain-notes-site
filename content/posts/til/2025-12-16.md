## 今天学到

搭建了一套「私有笔记 → 公开博客」的完整工作流，以后写东西终于有地方放了～

![[obsidian-quartz-workflow.jpg]]

## 背景

脑子里的想法太多太杂，需要一个既能私密记录、又能选择性公开的平台。Obsidian 本地写作体验一流，Quartz 静态博客够轻量，完美！

## 核心配置

| 组件 | 方案 | 一句话点评 |
|------|------|------------|
| 私有笔记 | `Rain-notes` (Private) | 乱七八糟的想法都丢这里 |
| 公开博客 | `Rain-notes-site` (Public) | 整理好的才会发布 |
| 编辑器 | Obsidian 双仓库模式 | 一个软件管两个库，舒服 |
| 部署 | Cloudflare Pages | 免费 + 全球 CDN |

## 关键命令

```bash
# 本地预览（写完先看看效果）
npx quartz build --serve

# Cloudflare 构建命令（这个踩了坑）
git fetch --unshallow && npx quartz build
```

## 踩坑记录

- Quartz 输出目录是 `public/`，不是 `dist/`（被坑了 10 分钟）
- Cloudflare 默认浅克隆，导致文章日期全乱了，加 `git fetch --unshallow` 解决
- 记得把构建输出目录改成 `public`，不然部署一片空白 😅
