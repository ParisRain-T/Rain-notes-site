# 路径索引

> 你只需要一条“能走下去”的路线，不需要一次看完所有知识。

## 如果你现在很混乱（先按这个重置）

1. 只打开 1 个文件：[[执行路径]]（选“本周 Gate”，别往后翻）
2. 只做 1 件事：把本周 Gate 做成“可演示 + 可验收”
3. 只有当你遇到具体问题，才打开对应专题页（例如协议对齐就看 [[P0_SPEC_v1]]）

> 规则：任何时候你打开的文档不要超过 3 个；其余都先关掉。

## 什么是学习路径？

数字花园里的笔记往往是碎片化的，但学习本身需要有先后顺序。学习路径就是把这些碎片**按主题串起来**，形成一条可以从头走到尾的阅读线索。

## 为什么要做路径？

1. **对读者友好**：告诉他们从哪开始、怎么深入
2. **展示思考深度**：不是零散的知识点，而是有体系的理解
3. **对面试有帮助**：面试官能快速看到你的学习脉络

1. 平时执行：只用 [[执行路径]]（本周 Gate → 交付物 → 验收）
2. 做实现对齐：遇到“对不上/改坏了”，回到 [[P0_SPEC_v1]]（单一真相）
3. 卡 bug：优先看 [[错误码与故障字典]]，配合 [[日志规范]] 与 [[测试体系]]
4. 复盘与投递：用 [[DoD清单]] 判定“完成”，用 [[作品集材料包]] 输出材料
